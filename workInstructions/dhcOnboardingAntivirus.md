## Introduction

### Purpose

Onboard a new customer with BDS as part of the VCS Antivirus service in accordance with Atos Global Delivery standards and portfolio services.

### Audience

- VCS Engineers

### Scope

That work instruction is intended to cover below tasks and activities:

1. Gather all prerequisite information.
2. Prepare Customer Information Sheet document (CIS).
3. Create a new request for VCS Customer onboarding within BDS service.
4. Validate and approve Service Costs.
5. Validate and Sign-off SO.

# Integration Steps

>**Note:** The print screens are illustrative and cannot be used as source for input data.

## Prerequisites

Prior to starting any onboarding related activities please note that following information would be required - gather all of them first.

- WBS code for internal WU-recharging
- contact TSM: (Name, email, phone)
- contact SDM: (Name, email, phone)
- contact Project/Transition (Name, email, phone)
- transition start date
- transition end date
- operating start date
- handover to shore date
- contract lifeline

## Prepare Customer Information Sheet

Use all information from previous step to fill the [Customer Information Sheet](files/dhcOnboardingAntiVirus/CustomerInformationSheet.docx) template.

>**Note:** Do not change any additional fields that are pre-defined/pre-selected.

## Create New Request

Once Customer Information Sheet is prepared please send an email to **<gabriela.gorzycka@atos.net>** to start the VCS onboarding process. Attach  Customer Information Sheet document.
Based on email new request for VCS Customer onboarding will be created.

>**Note:** Onboarding process requires approvals on BDS site as well as deployment team and can take up to 1-2 weeks.

## Validate and Approve Service Cost

After new request will be sent, BDS will get back and send an estimated cost calculated for amount of VCS servers. Cost should be validated and approved by Project. Once approved, it will be included in Service Offering document to charge Customer for Service.

## Validate and Sign-off SO

BDS is responsible for creating the Service Offering document. Once final version of SO is delivered, it should be validated. If no updates are required SO should be sign-off by both sites, BDS and VCS Project team, to fully finish onboarding process.
